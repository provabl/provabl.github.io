The Provabl Suite

Three tools, one compliance loop.

ground
Infrastructure Layer

Deploy a correctly-configured AWS organization foundation that attest can manage. Makes zero compliance claims — attest does that after attest scan.

  • Account structure: management, security, network, workload OUs
  • Transit Gateway, hub-and-spoke VPCs, VPC endpoints with org conditions
  • GuardDuty, Security Hub, Macie — all on by default
  • Permission boundaries that actually restrict (Deny-scoped)
  • Tagging SCPs with OR logic — every missing tag triggers deny

attest
Compliance Layer

Compile compliance frameworks into deployed policy artifacts. Enforce controls continuously. Generate audit evidence from live AWS state.

  • Framework support: CMMC L1/L2/L3, HIPAA, FedRAMP, NIH, ITAR, GDPR
  • Compiles to SCPs, Cedar policies, AWS Config rules
  • Continuous Cedar PDP enforcement via EventBridge
  • Generates SSP, OSCAL, SPRS, DMSP from live state
  • AI compliance navigator — surfaces obligations before they become violations

qualify
People Layer

Train and qualify researchers before granting access. Training completion writes IAM role tags that attest's Cedar policies evaluate in real time.

  • CUI, HIPAA, FERPA, ITAR, data classification training modules
  • Writes attest:* IAM tags on completion — Cedar-evaluable
  • Training expiry tracking with automated access revocation
  • NIH NOT-OD-26-017 research security training
  • Lab onboarding wizard with project context capture

How It Works Together

Each tool has one job. Together they close the compliance loop.

bash
# Step 1 — ground deploys correct foundations
ground deploy --config ground.yaml
✓ Org structure, network, identity, logging, security baseline provisioned ✓ Permission boundaries enforced, VPC endpoint policies org-scoped
# Step 2 — attest discovers and compiles
attest init --region us-east-1
attest frameworks add cmmc-level-2 hipaa
attest compile --scp-strategy merged
attest apply --approve
✓ 47 controls compiled → SCPs deployed, Cedar policies active
# Step 3 — qualify gates access by training
# researcher completes CUI + HIPAA training in qualify
✓ IAM role tagged: attest:cui-training=true, attest:hipaa-training=true ✓ Cedar PDP re-evaluates — access granted automatically
# Step 4 — attest makes the compliance claim
attest scan
attest generate ssp --framework cmmc-level-2
✓ Compliant (43/47 controls enforced, 4 operational) ✓ SSP generated from live AWS state — not manually assembled

Design Principles

Built for SREs and compliance officers who can't afford surprises.

Separation of claims

ground makes zero compliance claims. It deploys correct foundations. attest makes the compliance claim — but only after attest scan on live state.

Policies are tested

Every IAM boundary, SCP, and Cedar policy ships with unit tests. Permission boundaries are verified to deny privilege escalation. VPC endpoint policies are verified to have org conditions.

Live state, not documentation

Evidence is generated from what's actually deployed — not from what someone wrote in a doc. SSPs, OSCAL, SPRS reflect the real posture at scan time.

Training gates access

qualify writes IAM tags on training completion. attest's Cedar policies evaluate those tags before granting data access. No training = no access, enforced at the policy level.

Open core

Compilers, policy schemas, CLI tools, and framework definitions are open source. Commercial features (AI navigator, multi-SRE management, GRC integrations) support ongoing development.

Proactive, not reactive

The AI navigator surfaces obligations before they become violations. Training expiry, DUC renewals, CMMC assessment windows — flagged 30–90 days out, not the day they expire.
